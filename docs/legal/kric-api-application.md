## 3. 신청 목적

### 왜 KRIC API 데이터가 필요한가
시티드 v1.0은 서울교통공사 운영 1~9호선 + 광역철도(KORAIL 운영) 일부 구간을 다루는데, **공공데이터포털(data.go.kr) API만으로는 KORAIL 운영 광역철도 구간의 역사 정보가 부족**합니다. 구체적으로 다음 노선의 외곽 구간 역 정보가 필요합니다.

- **4호선 안산선 구간** (한대앞–오이도, KORAIL 운영)
- **1호선 경인선 구간** (KORAIL 운영, 인천 방면)
- **수인분당선 전 구간** (KORAIL 운영)
- **경의·중앙선 전 구간** (KORAIL 운영)

서울교통공사가 운영하는 도심 구간은 기존 API로 커버되지만, KORAIL이 운영하는 광역철도 외곽 구간은 **KRIC의 “전국 철도역 정보” 데이터셋이 정확한 표준 명칭·역 코드·노선 분류를 제공**하기 때문에 신청 드립니다.

### v1.1 확장 계획
v1.1 업데이트에서 광역철도 노선을 정식 추가 예정이며, KRIC API 데이터는 다음 화면에서 활용됩니다.
1. 역 검색 화면의 광역철도 역 목록 표시
2. 노선별 필터(예: “수인분당선” 필터)
3. 환승역 안내 시 호선/노선명 정규화

## 5. 데이터 활용 방식

### 어떻게 쓰는가
- 앱 내 **역 목록·검색·노선 필터** 표시 용도로만 사용
- 데이터는 클라이언트(이용자 기기)에서 표시하기 위해 앱 빌드 시 정적으로 묶거나, 실행 중 KRIC API를 직접 호출

### 어떻게 안 쓰는가
- **제3자에게 제공·재판매·공유하지 않습니다.**
- 광고 타겟팅, 마케팅, 통계 분석 등 본래 목적 외 활용은 하지 않습니다.
- 가공·결합해서 별도의 데이터 상품을 만들지 않습니다.

### 출처 표기
앱 내 “정보” 또는 라이선스 페이지에 KRIC 출처를 명시할 예정입니다. 예시:
> 본 앱은 국가철도공단(KRIC, data.kric.go.kr)이 제공하는 철도역 정보를 활용합니다.
